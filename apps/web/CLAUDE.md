# CLAUDE.md — apps/web (фронтенд)

Фронтенд аукциона. Акцент — **перевозчик (carrier)** и **витрина живых торгов**. Общие принципы проекта — в корневом `../../CLAUDE.md`; здесь только то, что касается web.

## Стек и текущее состояние
- **Next.js 16** (App Router) + **React 19** + **Tailwind CSS 4**.
- TypeScript 5, `strict: true`, алиас `@/*` → `./src/*`.
- Пакетный менеджер — **pnpm**.
- Сейчас в `src/app/` — дефолтный скаффолд (`layout.tsx`, `page.tsx`, `globals.css`). Страницы ниже создаются по задачам M5.

## Команды (из `apps/web/`)
```
pnpm dev      # дев-сервер
pnpm build    # прод-сборка
pnpm start    # запуск собранного
pnpm lint     # eslint (eslint-config-next)
```

## Скоуп фронта (§11 ТЗ)
- **Только UI перевозчика.** UI шиппера — вне MVP (write-путь шиппера дёргается seed/генератором на бэке).
- Целевые страницы:
  - **Список лотов** (`/lots`) — SSR, быстрый first paint, простые фильтры.
  - **Страница живого лота** (`/lots/:id`) — **главная витрина**: текущая лучшая ставка, обратный отсчёт, история, форма ставки; всё обновляется по WebSocket.
  - **Мои ставки** (`/me/bids`) — дашборд перевозчика.
  - **Ops-экран** (`/ops`, опц.) — состояния саг + содержимое DLQ.
- Auth: вход как `carrier` (JWT с бэка).

## Библиотеки (вводятся в M5-01, ещё не в package.json)
- `socket.io-client` — подписка на канал лота (парность к серверному socket.io). Главный источник live-обновлений.
- `@tanstack/react-query` — серверное состояние, кэш, инвалидация на realtime-событиях.
- Tailwind уже подключён.

## Конвенции
- App Router (`src/app/`), серверные компоненты по умолчанию; `"use client"` — только где нужны хуки/WS/интерактив.
- Список лотов — SSR ради first paint; живой лот — клиентский, на WebSocket.
- Доступ к API — через типизированный клиент с JWT; URL API из env.
- react-query инвалидирует кэш по WS-событиям (`bid.placed`, `lot.closed`, ...), не поллит.

## Контракт с бэком
- REST: `POST /auth/login`, `GET /lots`, `GET /lots/:id`, `POST /lots/:id/bids` (нужен `Idempotency-Key`), `GET /lots/:id/bids`, `GET /me/bids`.
- WS: `/realtime` — подписка на канал лота, события `bid.placed`, `lot.opened`, `lot.closing`, `lot.closed`, `settlement.*`.
- Форма ставки: генерируй `Idempotency-Key` на клиенте; обрабатывай 409 (`too_low`/`closed`) дружелюбным UX. Аукцион **на понижение** — меньшая цена лучше.
- Обратный отсчёт реактивен к анти-снайп-продлению `closeAt` (приходит по WS).

# M3-06 — notification: идемпотентный мультиканальный консьюмер

**Майлстоун:** M3 (горячий путь)
**Зависимости:** M2-02, M3-01, M3-05
**Оценка:** M (0.5–1 день)

## Цель
Outbound-нотификации (§4): идемпотентный consumer на `notification.q` (`bid.placed`, `lot.opened`, `lot.closed`) с realtime-push и email-заглушкой.

## Объём работ
- Consumer на `notification.q` (`BaseConsumer` из M2-02, dedup из M3-01).
- Каналы: realtime-push (через realtime-мост M3-05) + email-stub (лог/таблица «отправленного»).
- Шаблоны событий → сообщений (открытие лота, перебитая ставка, закрытие, победа — победа/шиппер придёт из saga в M4).
- Идемпотентность по `messageId` (нотификации не двоятся — §5).

## Definition of Done
- На `bid.placed`/`lot.closed` генерируется нотификация ровно один раз даже при повторной доставке.
- Email-stub фиксирует «отправку» (наблюдаемо для ops/тестов).
- Покрыто integration-тестом (доставка + дедуп).

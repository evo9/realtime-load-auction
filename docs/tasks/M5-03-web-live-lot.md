# M5-03 — web: страница живого лота (WS, отсчёт, форма ставки) — витрина

**Майлстоун:** M5 (фронт) — главная витрина проекта
**Зависимости:** M5-01, M3-03, M3-05
**Оценка:** L (1.5 дня)

## Цель
Страница живого лота (§11): текущая лучшая ставка, обратный отсчёт, история, форма ставки — всё обновляется по WebSocket. Это то, что смотрят первым.

## Объём работ
- Данные: `GET /lots/:id` (детали + лучшая ставка), `GET /lots/:id/bids` (история).
- WS-подписка на канал лота: live-обновление лучшей ставки, истории, статуса (open→closing→closed), анти-снайп-продление отсчёта.
- Обратный отсчёт до `closeAt` (реактивен к продлению).
- Форма ставки: генерация `Idempotency-Key`, `POST /lots/:id/bids`, обработка 201/409 (`too_low`/`closed`) с понятным UX.
- Состояния: лот закрыт/сеттлится/победитель.
- Оптимистичный апдейт + сверка по WS-событию.

## Definition of Done
- Ставка одного клиента мгновенно видна другому через WS (без рефреша).
- Отсчёт корректно продлевается при анти-снайпе.
- 409-реджекты (хуже/закрыт) отображаются дружелюбно; идемпотентный повтор не плодит ставок.
- Это демонстрируемо «вживую» на двух вкладках.

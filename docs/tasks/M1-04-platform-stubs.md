# M1-04 — Платформенные модули-заглушки + проводка в AppModule

**Майлстоун:** M1 (каркас)
**Зависимости:** M1-02
**Оценка:** S (0.5 дня)

## Цель
Создать пустые, но корректно регистрируемые модули платформенного слоя, чтобы `AppModule` собирался в финальной форме (§9.1) и каждый последующий пакет наполнялся без рефакторинга проводки.

## Объём работ
- Скелеты модулей: `platform/redis`, `platform/messaging`, `platform/outbox`,
  `platform/idempotency`, `platform/scheduler` (каждый — `@Module` с экспортом будущих провайдеров-заглушек).
- Регистрация в `app.module.ts` в порядке из §9.1.
- Каждый модуль читает свою секцию конфига (хосты/порты) из `ConfigService`.
- `redis`-модуль: подключение `ioredis` + ping в healthcheck (минимально живой).

## Definition of Done
- `AppModule` собирается со всеми platform-модулями, приложение стартует.
- `GET /health` пингует Redis (и Postgres из M1-03).
- Чёткие границы пакетов: domain-модули нигде не импортируют внутренности platform, только публичные провайдеры.

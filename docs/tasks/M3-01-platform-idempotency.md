# M3-01 — platform/idempotency: API-интерсептор + дедуп консьюмеров

**Майлстоун:** M3 (горячий путь)
**Зависимости:** M2-01
**Оценка:** M (1 день)

## Цель
Идемпотентность на двух уровнях (§5): входящий API по `Idempotency-Key` и консьюмеры по `messageId`. Без сторонних библиотек.

## Объём работ
- `IdempotencyService`: `begin(key)` → `SET NX idem:{key}` с TTL (возврат кэша при повторе), `complete(key, result)` → сохранить сериализованный результат. Поддержка статусов in-progress/done (защита от гонки параллельных дублей).
- API-интерсептор: читает заголовок `Idempotency-Key`, оборачивает горячие POST; повтор возвращает закешированный ответ и статус-код.
- Consumer-хелпер `dedup.seen(messageId)`/`dedup.mark(messageId)` → `msg:dedup:{messageId}` SET NX (используется в `BaseConsumer` из M2-02).
- TTL'ы из конфига.

## Definition of Done
- Два одинаковых POST с одним ключом → одна доменная операция, второй отдаёт кэш.
- Параллельные дубли не выполняют операцию дважды (in-progress lock).
- Повторная доставка RMQ-сообщения с тем же `messageId` обрабатывается один раз (тест на BaseConsumer).

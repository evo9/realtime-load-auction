# M3-04 — bidding: query-путь (история ставок, мои ставки)

**Майлстоун:** M3 (горячий путь)
**Зависимости:** M3-03
**Оценка:** S (0.5 дня)

## Цель
Read-путь модуля bidding (§10): история ставок по лоту и дашборд-данные перевозчика. Query-хендлеры бьют прямо в репозиторий (§4.1).

## Объём работ
- `GET /lots/:id/bids` — история ставок лота (пагинация, сортировка по выгодности/времени).
- `GET /me/bids` — ставки текущего перевозчика (JWT), статус (лидирует/перебита/выиграна).
- `GetBidsHandler` + DTO; индексы под запросы (lotId, carrierId).

## Definition of Done
- Эндпоинты возвращают корректные данные с пагинацией.
- `/me/bids` отдаёт только ставки текущего carrier.
- Покрыто unit/e2e-тестами (supertest).

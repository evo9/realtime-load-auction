# M4-04 — Ops-видимость: состояния саг + содержимое DLQ

**Майлстоун:** M4 (saga)
**Зависимости:** M4-02, M4-03
**Оценка:** M (0.5–1 день)

## Цель
Наблюдаемость для демо и собеса (§11 ops-экран): API/данные по состояниям саг и DLQ. Фронт-экран — опционально в M5-05.

## Объём работ
- `GET /ops/sagas` — список `saga_instances` с фильтром по статусу/шагу/лоту.
- `GET /ops/dlq` — содержимое DLQ-очередей (через RMQ management API или passive-consume), счётчики.
- (Опц.) endpoint повторного запуска/реквью застрявшей саги или requeue из DLQ — за ролью/флагом.
- Защита эндпоинтов (роль/админ-флаг).

## Definition of Done
- Видны running/failed/compensated саги с деталями шага.
- Сообщения в DLQ перечисляются с причиной/счётчиком.
- Данные пригодны для ops-экрана M5-05.
